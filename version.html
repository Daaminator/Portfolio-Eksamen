Versionsstyring

Verstionsstyring holder styr på hvilke ændringer der er blevet foretaget og gør en bruger opmærksom på.

Git er et open source versionsstyringssystem som bruges til at holde styr på ændringer eller tilføjelser der foretages i filer eller applikationer. Git bruges ofte i større virksomheder for at reducere tidsforbruget ved at flere medarbejdere kan arbejde på samme projekt, på flere forskellige maskiner.

Der findes forskellige programmer som benytter sig af Git protokollen, vi har arbejdet med GitHub.

Github er en online platform som gør det muligt at oprette online repositories i skyen, for at gøre deling og arbejde af projekter nemmere og hurtigere.

GitHub fungerer som et slags socialt medie, hvor hardcore udviklere og kode entusiaster kan dele kode og idéer.

GitHub bygger på følgende begreber:


Repository

  • Er stedet hvor man lagrer sine filer. Det kan f.eks være i en lokal mappe eller en ekstern server.

    • Commit

  • Når man har foretaget ændringer i sit projekt og ønsker at gemme dem, laves der et commit. Når man foretager en commit i GitHub, er det muligt at tilføje kommentarer til det arbejde man har lavet. Ens tidligere commits vil altid være tilgængelige, og man vil kunne gå tilbage og se eventuelle ændringer man har lavet.

    • Pull Request

  • Pull bruges til at hente og integrere ændringer der er blevet foretaget på ens projekt. Hvis en kollega vælger at ændre tilføje noget til et projekt, er det vigtigt at ”pulle” hans ændringer, og integrere dem men sit dokument inden der arbejdes videre.

    • Branch

  • En branch er en alternativ version af den oprindelige kode fil. Det vil altså sige at man med en branch, kan arbejde med en kode, uden at det får konsekvenser for original filen. Efter ens arbejde i en branch er færdigt, kan man merge sin branch med sin master.

    • Clone

  • Bruges til at klone projekter der ligger tilgængelige på GitHub.

    • Merge

  • Bruges til at samle alle eventuelle branches med ændringer til en endelig samlet master fil, for at have et bedre og mere samlet overblik.

    Oprettelse af repository på GitHub

    Desktop version:

    Download og installer den nyeste version af GitHub fra GitHubs hjemmeside

    Når programmet er installeret, kan der nu oprettes et repository ved at bruge genvejen: Kontrol+N.

    1. Giv dit nye repository et navn og eventuelt en beskrivelse

    2. Bestem stien hvori dine GitHub filer bliver gemt . Det er vigtigt altid at gemme HTML og CSS filændringer i samme mappe, for at GitHub kan se hvilke ændringer der er blevet foretaget.

    3. Afkryds boksen der tilbyder at oprette projektet med en README fil.

    4. Vælg eventuel licens du ønsker skal gælde for dit projekt

    Når projektet er oprettet, og man eventuelt har klonet et projekt fra GitHub, kan man nu lave en branch ud fra den nedhentede kode og begynde at ændre i den som man vil.

    Man opretter en ny branch ved at trykke på ens current branch, som er master branchen, og trykke på new.


    Giv din nye branch et navn, og du kan nu begynde at arbejde på master filen, i form af en ny branch, uden at lave ændringer i den.

    Når man har lavet ændringer, og man ønsker at gemme dem og gøre dem tilgængelige for andre at se. Kan man nu vælge at bruge commit funktionen, til at publicere sine ændringer. Her er det muligt at tilføje kommentare.



    Det er en god idé at lave indrykninger i ens kode, for at gøre den mere læsevenlig og generelt skabe overblik ved at kunne se hvor et tag bliver startet og hvor det slutter. For at lave indryk bruges tabulatorknappen frem for mellemrum.
